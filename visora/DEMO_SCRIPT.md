| **1:30–1:50** | **3D + polish.** Jump to **3D Studio** or trigger **3D** from chat: show a **model preview / GLB**. “This is still **fal.ai**—2D campaigns *and* **3D product reality** in one stack.” Option: quick **`/tech`** cut—three APIs, one arrow—*five seconds max*. |
| **1:50–2:00** | **Close & track fit.** “That’s VISORA: **OpenAI** creates the brain, **Supabase** saves the memory, **fal.ai** creates the **visual and 3D business reality**. We’re built for **Best Use of fal**—thank you.” End on **Gallery** or **logo** card. |

### Recording tips

- **Rehearse once** with cold keys / warm keys so you know latency.  
- **Mute notifications**; use **1080p**, clear system font scaling.  
- If generation is slow, **pre-run** one session and **screen-capture** a second “live” take with a shorter prompt.  
- Show **network** or **/tech** only if time allows—judges value **visible fal output** first.

## Optional Q&A hints *(don’t read on video)*

- **Why not only OpenAI Images?** We standardize on **fal.ai** for ** Flux-class** speed, **batch** semantics, and **3D** endpoints behind one integration.  
- **Is data private?** API keys stay **server-side**; see Next.js API routes in `/app/api`.  
- **What if APIs are down?** Brand route and visuals use **documented fallbacks** so the demo still tells a story.
